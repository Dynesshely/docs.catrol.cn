欢迎使用 KitX

KitX 是一款以插件驱动为核心的工具箱软件，它提供了让用户自定义所需的工具套装的功能，达到更好的易用性。
本篇文章意在阐明一些基本要求，要点，以及一些问题

要求

开始

KitX.Core

任何能被 KitX 正确识别并加载的插件均需要引用 KitX.Core 这个类库。
尤其是其中的 IContract 接口

获取 KitX.Core 的途径

  1. 在 KitX 插件开发 QQ 群中： 518573921， 群文件中 KitX.Core 文件夹， 实时更新最新版 KitX.Core 。
  2. 从 NuGet 下载 （待上传，当前此途径不受支持）。

KitX.Core 类库版本带来的问题

按照 常青园晚 订立的规则，使用 KitX.Core 版本为 b 的 KitX 本体软件 可以正确识别并加载 引用为版本 b 的 KitX.Core 的插件。
即：插件使用的 Core 版本与 KitX 本体软件是双向绑定的，不存在向后兼容，但是存在破坏性更新的问题
例如：使用 v1.2.8 版本 KitX.Core 的插件可以被 使用 v1.2.8 版本 KitX.Core 的 KitX 识别并加载

但是请注意：KitX.Core 的版本 与 KitX 的版本之间没有必然联系

GitHub

Github上是最新的源代码，地址为：KitX.Core-Github

Github上一般每天都有更新，相对而言不适合用于生产。
文档的更新速度必然跟不上最新的Github源码，有出入的地方一切以Github上Demo示例为准。
